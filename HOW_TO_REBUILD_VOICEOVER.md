# Rebuild the walkthrough with a natural, human-sounding voice

The original `create_voiceover_video.ps1` uses the old Windows SAPI voice
("Microsoft Zira"), which sounds robotic. `build_voiceover_natural.py` replaces
just the voice with Microsoft's free **neural** voices (Aria, Guy, Jenny, Ryan…),
which sound like a real person. It reuses your existing slide images and only
regenerates the narration and the video.

## One-time setup

1. **Python 3.8+** — https://www.python.org/downloads/ (check "Add Python to PATH").
2. **ffmpeg** on PATH — https://www.gyan.dev/ffmpeg/builds/ (the
   `ffmpeg-release-essentials` zip; add its `bin` folder to PATH).
3. **edge-tts**:
   ```powershell
   python -m pip install edge-tts
   ```
   (The script will offer to install it automatically if it's missing.)

> Internet is required *while it runs* — the neural voices are generated by
> Microsoft's online service. No account, no API key, free.

## Run it

From the project folder:

```powershell
python build_voiceover_natural.py
```

That produces:

```
media/DiskSpaceMonitor_v1_3_feature_walkthrough_natural.mp4
```

## Pick a different voice

```powershell
python build_voiceover_natural.py --voice en-US-GuyNeural      # relaxed male
python build_voiceover_natural.py --voice en-US-JennyNeural    # clear female assistant
python build_voiceover_natural.py --voice en-GB-RyanNeural     # British male narrator
python build_voiceover_natural.py --list-voices                # see them all
```

Default is `en-US-AriaNeural` (warm, friendly — good for tutorials).

Slow it down or speed it up:

```powershell
python build_voiceover_natural.py --rate -8%
```

## What changed vs. the old script

- **Voice:** neural (Aria/Guy/Jenny/Ryan) instead of SAPI "Zira".
- **Script:** the narration was rewritten to sound conversational — contractions,
  shorter clauses, natural pauses — so it doesn't read like a manual.
- **Timing:** each slide is held for exactly the narration length plus a short
  pause, so there are no frozen gaps after the voice stops.
- Slides are untouched; the original SAPI script and outputs are left in place.

## Why this didn't run inside the Claude sandbox

The sandbox's network is restricted to package repositories, so Microsoft's
voice endpoint (`speech.platform.bing.com`) is blocked there. It is **not**
blocked on your own Windows machine, which is why this runs locally.
